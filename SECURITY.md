# Security Policy

MemBrain is a self-hosted AI safety gateway. We take the security of the project —
and of the deployments that run it — seriously.

## Reporting a vulnerability

Please report security vulnerabilities **privately**. Do not open a public issue
for a suspected vulnerability.

- **Preferred:** use GitHub's private vulnerability reporting — the **"Report a
  vulnerability"** button on this repository's
  [Security tab](https://github.com/mrpintcom/membrain-engine/security).
- **Alternatively:** reach us through the contact options on
  [membrn.ai](https://membrn.ai).

Please include a description, reproduction steps, the affected version, and the
impact. We aim to acknowledge reports within **3 business days** and will keep you
updated as we investigate and ship a fix.

## Supported versions

MemBrain is pre-1.0 (currently the **v0.10.x** line). Security fixes target the
latest released version — please reproduce on the latest build before reporting.

## Scope

This policy covers the MemBrain gateway distributed from this repository. Because
MemBrain is **self-hosted**, the security of any given deployment also depends on
its configuration, secrets management, and surrounding infrastructure — review the
deployment and security guidance in the docs before relying on it for regulated
data flows.

## Disclosure

We practice coordinated disclosure: we will work with you on a reasonable timeline
to remediate and disclose, and we're glad to credit reporters who wish to be
acknowledged.
